缘起

今天接到一个任务：读取一份儿童基因检测 PDF 报告，然后制作一份销售培训 PPT。听起来是个体力活，但正好可以测试一下自动化能力。

目标很明确：用 Python 实现 PDF 内容提取和 PPT 自动生成。

---

技术栈选择

PDF 读取方案对比

| 库 | 优点 | 缺点 | 适用场景 | |----|------|------|---------| | PyPDF2 | 安装简单，速度快 | 表格提取弱 | 纯文本 PDF | | pdfplumber | 表格提取强 | 依赖多，安装慢 | 含表格 PDF | | PyMuPDF | 功能最强 | 许可证限制 | 复杂 PDF |

最终选择了 PyPDF2，因为这份 PDF 主要是文字内容，没有复杂表格。

PPT 生成方案

| 方案 | 优点 | 缺点 | |------|------|------| | python-pptx | 纯 Python，灵活 | 不支持渐变/阴影 | | Marp | Markdown 转 PPT | 样式固定 | | 在线 AI 工具 | 设计精美 | 需要手动操作 |

选择了 python-pptx，虽然不支持高级效果，但可以通过技巧模拟。

实战记录

第一步：安装依赖

``bash pip3 install PyPDF2 pip3 install python-pptx `

两个库都安装顺利，没有遇到依赖问题。

第二步：PDF 内容提取

核心代码：

`python import PyPDF2

pdf_path = '/path/to/report.pdf'

with open(pdf_path, 'rb') as f: reader = PyPDF2.PdfReader(f) print(f'总页数：{len(reader.pages)}') for i, page in enumerate(reader.pages): text = page.extract_text() if text: print(f'=== 第{i+1}页 ===') print(text[:2000]) # 限制输出长度 `

执行结果：成功提取了 35 页 PDF 的内容，中文显示正常。

关键点：

  • 使用 rb 二进制模式打开
  • 中文编码自动处理
  • 支持批量提取多页

    • 第三步：PPT 基础结构

    `python from pptx import Presentation from pptx.util import Inches, Pt

    prs = Presentation() prs.slide_width = Inches(13.333) # 16:9 prs.slide_height = Inches(7.5)

    添加空白幻灯片

    slide = prs.slides.add_slide(prs.slide_layouts[6])

    添加标题

    title_box = slide.shapes.add_textbox( Inches(0.5), Inches(0.5), Inches(12), Inches(1) ) tf = title_box.text_frame p = tf.paragraphs[0] p.text = "标题" p.font.size = Pt(36) p.font.bold = True

    保存

    prs.save('output.pptx')     `

    第四步：卡片式布局

    为了实现美观的卡片效果，我封装了一个函数：

    `python from pptx.dml.color import RGBColor from pptx.enum.shapes import MSO_SHAPE

    def add_card(slide, title, points, left, top, w, h): # 卡片背景（圆角矩形） card = slide.shapes.add_shape( MSO_SHAPE.ROUNDED_RECTANGLE, left, top, w, h ) card.fill.solid() card.fill.fore_color.rgb = RGBColor(255, 255, 255) card.line.width = Pt(2) card.line.color.rgb = RGBColor(41, 98, 255) # 蓝色边框 # 标题 title_box = slide.shapes.add_textbox( left+Inches(0.3), top+Inches(0.25), w-Inches(0.6), Inches(0.5) ) tf = title_box.text_frame p = tf.paragraphs[0] p.text = title p.font.size = Pt(18) p.font.bold = True p.font.color.rgb = RGBColor(41, 98, 255) # 内容 content_box = slide.shapes.add_textbox( left+Inches(0.3), top+Inches(0.85), w-Inches(0.6), h-Inches(1) ) tf2 = content_box.text_frame for i, point in enumerate(points): if i == 0: p = tf2.paragraphs[0] else: p = tf2.add_paragraph() p.text = "• " + point p.font.size = Pt(14) `

    第五步：配色方案

    `python

    主题色

    PRIMARY = (41, 98, 255) # 科技蓝 SECONDARY = (0, 184, 148) # 青绿色 LIGHT = (230, 240, 255) # 浅蓝背景 DARK = (45, 52, 54) # 深灰文字 WHITE = (255, 255, 255) # 白色 `

    选择理由：蓝色系适合科技/医疗主题，青绿色作为点缀色增加活力。

    踩坑记录

    坑 1：中文文件名

    保存 PPT 时，如果文件名包含中文，需要用字符串而不是 Path 对象：

    `python

    ❌ 可能出错

    prs.save(Path('输出.pptx'))

    ✅ 正确

    prs.save('输出.pptx')     ``

    坑 2：文本框溢出

    如果文字太多，文本框会溢出。解决方法：

  • 限制每页内容量
  • 使用自动换行
  • 动态调整文本框高度

    • 坑 3：渐变不支持

    python-pptx 真的不支持渐变填充，只能用多层矩形模拟。效果还行，但代码复杂了不少。
