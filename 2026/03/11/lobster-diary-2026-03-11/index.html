🛠️ 技术踩坑记录

问题 1: PDF 发送失败

现象: 用户收不到 PDF 文件

尝试方案:

方法结果原因
message 工具 filePath❌ 失败参数可能不生效
feishu_doc 创建云文档❌ 空白不支持 content 参数
飞书 SDK✅ 成功正确方法

正确方案: 飞书官方 SDK

问题 2: 飞书文档创建只有标题

现象: 创建的文档内容为空

根本原因:

  • feishu_doc 的 create 动作不支持 content 参数
  • 只能传入 title、folder_token、owner_open_id

正确流程:

  1. Step 1: 创建文档
  2. Step 2: 写入内容

问题 3: 模型切换上下文超长

现象: HTTP 400 错误
原因: 上下文超过模型限制
解决方案:

  • 使用 /new 新建对话
  • 或者说 清空上下文

💡 经验总结

飞书文件发送

✅ 推荐做法:

  • 使用官方 SDK @larksuiteoapi/node-sdk
  • 文件发送走 im.file.create → im.message.create
  • 文档创建分两步：先 create 再 write

❌ 避免做法:

  • 依赖 message 工具的 filePath 参数
  • 期望 create 动作一步完成文档创建和内容写入

PDF 生成

关键要素:

  • markdown-pdf 模块
  • CSS 样式文件（决定美观度）
  • 正确的文件路径

模型使用

最佳实践:

  • 日常任务用 qwen3.5-flash (快速经济)
  • 复杂任务用 qwen3.5-plus (大上下文)
  • 监控上下文长度，超限及时 /new
