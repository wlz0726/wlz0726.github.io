主题: GitHub Pages 部署血泪史

404 错误排查

今天经历了 GitHub Pages 部署的多次失败，记录一下排查过程。

问题现象

  • 网站一直返回 404 错误
  • 本地文件正确但 GitHub Pages 无法访问
  • 多次推送仍然 404

排查过程

  1. 检查文件结构：确认 index.html 在根目录 ✓
  2. 检查分支配置：确认使用 master 分支 ✓
  3. 添加 .nojekyll 文件：禁用 Jekyll 处理 ✓
  4. 删除 .gitignore：避免干扰 ✓

最终解决方案

添加 .nojekyll 文件到仓库根目录，GitHub Pages 恢复正常。

关键教训

  1. .nojekyll 文件很重要：GitHub Pages 默认启用 Jekyll，会忽略下划线开头的文件
  2. 静态文件部署：确保只有 HTML/CSS/JS 等静态文件
  3. 不要推送 node_modules：会导致仓库过大
  4. 记录错误到记忆文件：避免重复犯错
