## 缘起

今天正式开始了我的 AI Agent 养成计划。选择的是 OpenClaw 框架，一个本地优先的 Agent 系统。安装过程出乎意料地顺利，`npm install -g openclaw` 一条命令搞定。

---

## 核心工作：模型配置

### 配置文件结构

配置文件位于 `~/.openclaw/openclaw.json`，主要包含两大部分：

```json
{
  "models": {
    "providers": {
      "codingplan": {
        "qwen3.5-plus": {
          "contextWindow": 1048576,
          "maxTokens": 8192
        }
      }
    }
  },
  "agents": {
    "defaults": {
      "model": "qwen3.5-plus"
    }
  }
}
```

**关键参数解释**：
- `contextWindow`: 上下文窗口大小（单位：tokens）
- `maxTokens`: 最大输出长度
- `providers`: 模型仓库，可以有多个供应商

### 模型选择策略

经过一番研究，我确定了以下配置策略：

| 场景 | 模型 | 上下文 | 输出 |
|------|------|--------|------|
| 日常对话 | qwen3.5-plus | 1M | 8k |
| 编程任务 | qwen3-coder-plus | 1M | 64k |
| 快速响应 | qwen3.5-flash | 30k | 4k |

**选择理由**：
- `qwen3.5-plus`: 平衡性能和成本，适合日常使用
- `qwen3-coder-plus`: 针对代码优化，输出上限高
- `qwen3.5-flash`: 轻量级，适合简单任务

## 技术突破

### 1. 配置热重载

最初修改配置后需要重启整个服务，后来发现了正确的重载方式：

```bash
# 方式 1：重启 Gateway
openclaw gateway restart

# 方式 2：会话内切换模型
/model qwen3.5-plus
```

**经验教训**：修改配置前先备份，重载失败时可以快速恢复。

### 2. 流式输出

开启流式输出后，响应是逐字显示的，大大提升了交互的即时感。这个选项在配置文件中默认开启，但值得确认一下。

### 3. 紧急制动

学会了一个保命命令：

```bash
/stop
```

当 Agent 开始执行不合适的操作时，可以立即中断。这个命令应该放在随手可及的地方。

---

## 踩坑记录

### 坑 1：模型别名混淆

配置文件中可以设置模型别名，但不同地方的别名可能不通用。最佳实践是始终使用完整模型名 `provider/model-name`。

### 坑 2：token 消耗预估

1M 上下文听起来很大，但实际使用中要注意输出 tokens 也会计入成本。设置合理的 `maxTokens` 很重要。

---

## 核心收获

1. **配置即教育**：配置文件不仅是参数设置，更是对 Agent 的"家庭教育"
2. **模型选择是权衡**：没有最好的模型，只有最适合场景的模型
3. **热重载很重要**：避免频繁重启服务，提升开发效率
