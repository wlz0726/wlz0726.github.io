# 🦞 龙虾日记 #2

**日期**: 2026-03-03  
**主题**: 语音功能探索与飞书集成

---

## 缘起

昨天配置好了基础模型，今天想给 Agent 加上语音能力。想象一下，如果它能用声音回复消息，而不只是冷冰冰的文字，交互体验会提升一个维度。

目标很明确：实现文字 → 语音 → 飞书消息的完整链路。

---

## 技术栈选择

### 语音合成方案对比

| 方案 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| 百度 TTS | 中文效果好 | 需要 API Key | 免费额度有限 |
| 阿里云 TTS | 音质好 | 配置复杂 | 按量付费 |
| edge-tts | 免费、简单 | 依赖网络 | 免费 |

最终选择了 **edge-tts**，理由很简单：免费、安装简单、中文语音质量不错。

### 完整工具链

```
文字 → edge-tts → MP3 → ffmpeg → Opus → 飞书 API → 语音消息
```

**为什么需要转码？** 飞书要求语音消息必须是 Opus 格式，而 edge-tts 输出的是 MP3。

## 实战记录

### 第一步：安装 edge-tts

```bash
/Users/wlz/anaconda3/bin/python3 -m pip install edge-tts
```

**注意**：需要使用 Python 3.10+ 版本，系统自带的 Python 3.9.6 会有兼容性问题。

### 第二步：测试 TTS 生成

```bash
/Users/wlz/anaconda3/bin/python3 -m edge_tts \
  --voice zh-CN-YunxiaNeural \
  --text "你好世界" \
  --write-media /tmp/voice.mp3
```

执行成功后，`/tmp/voice.mp3` 文件就生成了。用播放器试听了一下，效果出乎意料地好，声音自然，没有明显的机械感。

### 第三步：探索可用语音

```bash
edge-tts --list-voices
```

输出结果很长，我整理了几种常用的中文语音：

| 语音 | 特点 | 适用场景 |
|------|------|---------|
| zh-CN-YunxiaNeural | 男声，可爱卡通 ⭐ | 默认推荐 |
| zh-CN-YunxiNeural | 男声，阳光活力 | 营销内容 |
| zh-CN-XiaoxiaoNeural | 女声，亲切温柔 | 客服场景 |
| zh-CN-YunjianNeural | 男声，激情体育 | 运动内容 |
| zh-CN-liaoning-XiaobeiNeural | 女声，东北话 | 幽默内容 |
| zh-CN-shaanxi-XiaoniNeural | 女声，陕西话 | 方言内容 |

最后默认选择了 `YunxiaNeural`，声音比较中性，适合日常使用。

### 第四步：转码 Opus

这是今天最大的坑。飞书要求语音消息必须是 Opus 格式，采样率 16kHz，单声道。

```bash
ffmpeg -y -i /tmp/voice.mp3 \
  -c:a libopus \
  -b:a 32k \
  -ar 16000 \
  -ac 1 \
  /tmp/voice.opus
```

**参数解释**：
- `-c:a libopus`: 使用 Opus 编码器
- `-b:a 32k`: 比特率 32kbps
- `-ar 16000`: 采样率 16kHz
- `-ac 1`: 单声道

### 第五步：获取语音时长

飞书 API 需要语音时长（毫秒），用 ffprobe 获取：

```bash
ffprobe -v error \
  -show_entries format=duration \
  -of default=noprint_wrappers=1:nokey=1 \
  /tmp/voice.opus
```

输出类似 `1.903`，乘以 1000 就是毫秒数。

## 踩坑记录

### 坑 1：Python 版本问题

一开始用系统自带的 Python 3.9.6 安装 edge-tts，运行时提示 `typing` 模块兼容性问题。后来改用 anaconda3 的 Python 3.10 才解决。

**教训**：检查 Python 版本，确保 3.10+。

### 坑 2：ffmpeg 安装缓慢

`brew install ffmpeg` 编译时间超长，等了快半小时。后来发现是因为没有预编译 bottle，需要从源码编译。

**建议**：提前安装 ffmpeg，不要等到用时才装。

### 坑 3：飞书 API 格式要求

第一次发送时用了 `msg_type: "file"`，结果飞书显示为文件而不是语音消息。正确应该是 `msg_type: "audio"`，并且 content 里只需要 `file_key`。

**正确格式**：
```json
{
  "msg_type": "audio",
  "content": "{\"file_key\": \"file_v3_xxx\"}"
}
```

## 核心代码

### Node.js 发送语音（飞书 SDK）

```javascript
const { Client } = require("@larksuiteoapi/node-sdk");
const fs = require("fs");

const client = new Client({
  appId: "cli_xxx",
  appSecret: "xxx",
});

// 上传文件
const uploadRes = await client.im.file.create({
  data: {
    file_type: "opus",
    file_name: "voice.opus",
    file: fs.createReadStream("/tmp/voice.opus"),
    duration: 1903, // 毫秒
  },
});

// 发送消息
const sendRes = await client.im.message.create({
  params: { receive_id_type: "open_id" },
  data: {
    receive_id: "ou_xxx",
    msg_type: "audio",
    content: JSON.stringify({ file_key: uploadRes.file_key }),
  },
});
```

## 核心收获

1. **edge-tts 是真香**：免费、简单、音质好，强烈推荐
2. **格式转换是常态**：不同平台有不同的格式要求，ffmpeg 是必备工具
3. **API 文档要细读**：飞书的语音消息格式和直觉不一样，必须仔细看文档
